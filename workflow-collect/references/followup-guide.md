# Demo & Follow-up Guide (폴로업 가이드)

이 문서는 MVP Demo 시연 및 1주일 후 폴로업 프로세스 가이드입니다.

## 전체 타임라인

```
[Day 0] 인터뷰
    ↓
[Day 0-1] JTBD 정리 + Go/No-Go 판단
    ↓
[Day 1-7] MVP 개발
    ↓
[Day 7] Demo 시연
    ↓
[Day 14] 1주일 후 폴로업
    ↓
[Ongoing] Iteration 또는 Pivot
```

---

## Phase 1: Demo 시연 (Day 7)

### Demo 준비

#### 1-1. Demo 일정 잡기

**타이밍**: MVP 개발이 80% 완료되었을 때 (Day 5-6)

**메시지 템플릿**:
```
[이름]님, 안녕하세요!

말씀해주신 [워크플로우 이름]에 대한 MVP가 거의 완성되었습니다.

이번 주 내로 Demo를 보여드리고 싶은데,
30분 정도 시간 내주실 수 있으실까요?

가능하신 날짜/시간대 알려주시면 맞춰볼게요!
```

---

#### 1-2. Demo 자료 준비

**준비물**:
1. **데모 시나리오**: 인터뷰이가 언급한 pain point를 해결하는 시나리오
2. **테스트 데이터**: 실제 데이터와 유사한 샘플 데이터
3. **Before/After 비교**: 기존 방식 vs 새로운 MVP

**Demo 시나리오 예시**:
```markdown
## Demo Scenario

### Before (기존 방식)
1. Excel 파일 열기 (2분)
2. 데이터 수집 (30분)
3. 수동 정제 (1시간)
4. 분석 및 리포트 작성 (30분)
**Total: 2시간**

### After (MVP 사용)
1. MVP 열기 (10초)
2. 파일 업로드 (10초)
3. "지난 주 성과 요약해줘" 입력 (5초)
4. 결과 확인 (1분)
**Total: 2분**

### Key Improvement
- **시간 절약**: 2시간 → 2분 (60배 빠름)
- **오류 감소**: 수동 작업 제거로 오류율 0%
- **새로운 가능성**: 자연어 질의로 다양한 분석 가능
```

### Demo 진행

#### 2-1. Demo 구조 (30분)

**1. Opening (5분)**:
- 간단한 인사
- "지난번 인터뷰에서 말씀하신 [pain point]를 해결하기 위해 만들었습니다"
- Demo 순서 안내

**2. Before 회고 (5분)**:
- "지난번 인터뷰에서 이런 프로세스로 하신다고 하셨죠?"
- 기존 방식 간단히 리마인드
- Pain point 재확인

**3. Demo 시연 (10분)**:
- 실제로 사용하는 모습 보여주기
- 인터뷰이의 실제 use case 시연
- 주요 기능 설명

**4. 질문 & 피드백 (5분)**:
- "어떻게 보이시나요?"
- "실제로 사용해보시겠어요?"
- 피드백 수집

**5. 전달 & 다음 스텝 (5분)**:
- 접근 방법 안내 (URL, 설치 방법 등)
- 1주일 후 폴로업 일정 잡기
- 감사 인사

---

#### 2-2. Demo 중 체크포인트

**인터뷰이 반응 관찰**:

| 반응 | 의미 | 다음 액션 |
|------|------|----------|
| "와, 이거 진짜 좋은데요!" | ✅ 긍정적 | → 바로 사용법 안내 |
| "오... 괜찮네요" | ⚠️ 미지근 | → "어떤 부분이 기대와 다른가요?" 질문 |
| "음... 제가 원한 건 좀 다른데..." | ❌ 부정적 | → 기대와 차이 파악 |

**즉시 질문**:
- "이게 실제 업무에 도움이 될 것 같으세요?"
- "1주일 동안 써보실 수 있으세요?"
- "개선했으면 하는 부분이 있나요?"

#### 2-3. Demo 결과 분류

Demo 직후, 인터뷰이의 열정도를 평가합니다.

##### 🔥 열정적 (Big Fan 가능성 높음)

**특징**:
- "이거 진짜 좋다", "동료한테 소개해도 되냐"
- 즉시 사용법을 배우려고 함
- 추가 기능을 요청

**다음 액션**:
- Iteration 투자 고려
- Testimonial 요청 예정
- 해당 직무를 Public 채널 우선순위로

##### 😐 미지근 (Pivot 고려)

**특징**:
- "괜찮네요", "한번 써볼게요"
- 크게 흥분하지 않음
- 질문이 별로 없음

**다음 액션**:
- 1주일 후 폴로업에서 재평가
- 사용하지 않으면 빠르게 Pivot

##### 😞 부정적 (실패)

**특징**:
- "제가 원한 건 이게 아닌데..."
- 기능이 기대와 다름
- 사용하지 않겠다는 의사 표시

**다음 액션**:
- 피드백 수집 후 Learning으로 기록
- Pivot (다른 인터뷰이로 이동)

|------|----------|----------|

## Phase 2: 1주일 후 폴로업 (Day 14)

### 폴로업 목적

1. **사용 여부 확인**: 실제로 사용했는가?
2. **확산 여부 확인**: 주변에 자랑했는가? (Big Fan & Scalable 검증)
3. **Iteration 판단**: 추가 개선이 필요한가, Pivot해야 하는가?

---

### 폴로업 진행

#### 4-1. 폴로업 메시지

**타이밍**: Demo 후 정확히 7일 뒤

**메시지 템플릿**:
```
[이름]님, 안녕하세요!

지난주에 전달드린 [MVP 이름] 사용해보셨나요?

15-20분 정도 간단히 통화로 사용 경험을 여쭤볼 수 있을까요?
- 실제로 사용하셨는지
- 어떤 점이 좋았고 불편했는지
- 혹시 주변 분들께 소개하셨는지

가능하신 시간 알려주시면 맞춰볼게요!
```

---

#### 4-2. 폴로업 인터뷰 구조 (15-20분)

**1. 사용 여부 확인 (5분)**

**Q1**: "지난 일주일 동안 [MVP]를 사용해보셨나요?"

##### Case 1: 사용했음 ✅

**Q1-1**: "몇 번 정도 사용하셨나요?"
- 매일? → **Big Fan 신호**
- 2-3번? → 관심 있음
- 1번? → 그냥 테스트만

**Q1-2**: "실제 업무에 도움이 되었나요?"
- "네, 시간을 많이 절약했어요" → **High Impact**
- "괜찮았어요" → Medium Impact
- "생각보다는..." → Low Impact

**Q1-3**: "이전 방식으로 다시 돌아갈 수 있을 것 같으세요?"
- "안 돌아갈 것 같아요" → **Big Fan 신호**
- "둘 다 괜찮을 것 같아요" → Medium
- "기존 방식도 나쁘지 않았어요" → Low

##### Case 2: 사용하지 않음 ❌

**Q2-1**: "사용하지 않으신 이유가 있나요?"

**답변 유형별 대응**:

| 답변 | 의미 | 다음 액션 |
|------|------|----------|
| "바빠서 아직 못 써봤어요" | 우선순위 낮음 | → 2-3일 더 기다린 후 재확인 |
| "필요한 상황이 없었어요" | Pain point가 실제로는 크지 않음 | → Pivot |
| "사용법이 복잡해서..." | UX 문제 | → Iteration 가능성 |
| "우리 회사 정책상..." | 제약 조건 미파악 | → Learning, Pivot |

---

**2. 확산 여부 확인 (5분)**

**Q3**: "혹시 동료분들이나 주변 분들께 소개하셨나요?"

##### Case 1: 소개했음 ✅ (Big Fan & Scalable)

**Q3-1**: "누구에게 소개하셨나요?"
- 같은 직무 동료? → **직무 보편성 확인됨**
- 다른 팀? → 확장 가능성
- 외부 지인? → 강력한 Big Fan

**Q3-2**: "그분들 반응은 어떠셨나요?"
- "관심 있어 하던데요" → **잠재 고객**
- "자기들도 써보고 싶대요" → **Big Fan 확산**
- "별로 관심 없더라고요" → 타겟 미스매치

**Q3-3**: "혹시 그분들도 인터뷰해볼 수 있을까요?" (Indirect network 확장)

##### Case 2: 소개하지 않음 ❌

**Q4-1**: "왜 소개하지 않으셨나요?"

**답변 유형별 분석**:

| 답변 | 의미 | 판단 |
|------|------|------|
| "아직 충분히 안 써봐서..." | 확신 부족 | → 2주 더 사용 후 재평가 |
| "우리 팀만 쓸 수 있어서..." | 제약 조건 | → 확장 불가 (Learning) |
| "별로 자랑할 만한 건 아니라서..." | Impact 낮음 | → Pivot |
| "소개할 생각을 못 했어요" | 자연스럽게 안 떠올랐음 | → Medium Impact |

---

**3. 피드백 & Iteration 판단 (5-10분)**

**Q5**: "사용하시면서 불편한 점이나 개선했으면 하는 부분이 있나요?"

**피드백 유형별 대응**:

| 피드백 유형 | Iteration 가능성 | 예상 소요 시간 |
|------------|----------------|--------------|
| 버그 수정 | ✅ 높음 | 1-2일 |
| 기능 추가 (작은 것) | ✅ 높음 | 2-3일 |
| 기능 추가 (큰 것) | ⚠️ 조건부 | 1주 이상 |
| UX 개선 | ✅ 높음 | 2-3일 |
| 근본적 변경 | ❌ Pivot | N/A |

---

**4. Iteration 여부 결정**

인터뷰이의 **열정도**에 따라 결정합니다.

##### 🔥 열정적 (Big Fan) → Iteration 투자

**조건**:
- 매일 사용
- 주변에 자랑했음
- "이거 없으면 안 돼요" 수준

**액션**:
- 1주일 추가 투자하여 iteration
- Testimonial 요청
- 해당 직무를 Public 채널 우선순위로 설정

**메시지**:
```
피드백 감사합니다!
말씀하신 [기능/개선] 부분, 이번 주 내로 업데이트할게요.
정말 도움이 되셨다니 저도 기쁩니다 :)
```

---

##### 😐 미지근 → Pivot

**조건**:
- 1-2번만 사용
- 주변에 소개 안 함
- "괜찮긴 한데..." 수준

**액션**:
- 추가 투자 안 함
- Learning 기록
- 다음 인터뷰이로 Pivot

**메시지**:
```
피드백 감사합니다!
사용 경험 들려주셔서 많은 도움이 되었어요.
혹시 나중에 다른 불편한 점 있으시면 언제든 연락주세요!
```

---

##### 😞 부정적 → 빠르게 Pivot

**조건**:
- 사용 안 함
- "별로 도움 안 됐어요"
- "기대와 달랐어요"

**액션**:
- 즉시 Pivot
- Learning 기록 (왜 실패했는지)
- 다음 인터뷰이로 이동

**메시지**:
```
솔직한 피드백 정말 감사합니다!
기대에 못 미쳐서 죄송합니다.
다음에 더 나은 결과로 찾아뵙겠습니다 :)
```

- 새로운 통합

### Big Fan 패턴 분석

Big Fan이 여러 명 나오면, 공통 패턴을 찾습니다.

**분석 질문**:
1. 어떤 직무에서 Big Fan이 많이 나왔나?
