# MVP 제안서 템플릿

**Version**: 1.0
**Date**: 2026-02-04
**Purpose**: Phase 1 (JTBD) 완료 후 Phase 2에서 사용자에게 제시할 Impact 중심 MVP 제안서

이 문서는 JTBD 분석이 완료된 후, 인터뷰 대상자에게 제시할 MVP 제안서 템플릿입니다.

## 템플릿 사용 방법

1. **Phase 1 (JTBD)이 완료되어야 합니다**
   - AS-IS Workflow 문서화 완료
   - Top 3 Pain Points 식별 완료
   - AI 재설계 기회 분석 완료
   - GO 결정 완료

2. **JTBD 문서에서 정보 추출**
   - Pain Points → Section 1 (현재 상황)
   - TO-BE Workflow → Section 2 (해결 방법)
   - AI Redesign → Section 3-5
   - Follow-up Questions → Section 9 (미확인 사항)

3. **사용자 친화적 언어로 작성**
   - 기술 용어 최소화
   - Impact 중심 (시간 절약, 스트레스 감소 등)
   - 구체적 숫자와 예시 포함

4. **피드백 유도**
   - 각 섹션에 질문 포함
   - Out of Scope 명시적으로 제시
   - 성공 기준을 사용자가 정의하도록 유도

## 작성 가이드라인

### 톤 & 스타일
- **사용자 언어 사용**: 기술 용어보다 사용자가 이해하는 용어 사용
- **Impact 중심**: "어떻게"보다 "왜"와 "무엇을" 강조
- **구체적 숫자**: "많이 줄어듭니다"보다 "90% 감소"
- **감정 반영**: 인터뷰에서 나온 감정 표현 인용 ("진절머리 난다" 등)

### 피드백 유도
- **열린 질문 포함**: Yes/No가 아닌 생각을 유도하는 질문
- **예시 제공**: 사용자가 답변하기 쉽도록 예시 포함
- **선택지 제한**: 너무 많은 선택지는 피로도 증가 (2-4개 권장)

### Scope 조정 설명
- **시나리오별 분기**: 답변에 따라 어떻게 달라지는지 명확히
- **Critical 표시**: 기술적으로 블로킹되는 이슈는 명시적으로 표시
- **대안 제시**: 불가능한 경우 대안 방향 제시

### 시각적 요소
- **이모지 활용**: 섹션 제목에 이모지로 가독성 향상
- **표 활용**: Before/After 비교는 표로 시각화
- **코드 블록 활용**: UI 목업이나 플로우는 ASCII art로 표현

## 문서 저장 위치

완성된 MVP 제안서는 다음 위치에 저장합니다:

```
./P-S Tree/
├── [Business Function]/          # 직무별 폴더
│   ├── [Problem]/                # 문제별 폴더
│   │   └── [Solution].md         # MVP 제안서 (이 템플릿 사용)
│   └── ...
└── mvp-solutions.json            # P-S Tree 구조 추적
```

**예시**:
- `./P-S Tree/CEO & Founder/Accounting_Tax_Management/AI_Tax_Assistant.md`
- `./P-S Tree/Marketing/Campaign_Reporting/Auto_Report_Generator.md`
