# Opportunity Relevance Indicator - Ideation Session Log

**일시**: 2026년 1월 26일
**참여자**: PM
**주제**: Opportunity 리스트에서 Company Description과의 매칭 이유를 설명하는 피처

---

## 1. 문제 정의

### 배경
- [Opportunity 리스트 확인] → [Relevant Opportunity 선택] 전환율이 50%로 기대 이하
- 원인은 크게 두 가지로 구분:
  1. 실제로 적합한 Opportunity가 리스트에 없는 경우
  2. **실제로는 적합한 Opportunity가 있지만 유저가 판단하지 못하는 경우** ← 이번 피처의 타겟

### 현재 유저 행동
- 공고 하나당 3-5분 투자 필요 (Title에 구체적 정보 없어 Description 전체 읽어야 함)
- 전환 성공 유저는 10개 이상 확인
- Desktop 3개 / Mobile 1개만 한 화면에 표시

### 핵심 가설
적합한 공고가 있지만 유저가 Description을 심도있게 읽기 어려워 판단하지 못하고 있다.

---

## 2. 솔루션 탐색

### 초기 제안된 옵션
1. **본문 하이라이트**: Description 내 매칭 단어/문장에 형광색 표시
2. **"Why it's relevant"**: AI가 생성한 1-2문장 설명

### 추가 탐색된 옵션
- **매칭 태그**: Business Portfolio 아이템명을 태그로 표시
- **조합 옵션**: 태그 + Why it's relevant

### 최종 선택
**매칭 태그 + 본문 하이라이트 조합**

선택 이유:
- 태그: 5-10초 내 빠른 스캔 가능
- 하이라이트: 맥락 제공, semantic 연결 표현
- Why it's relevant: 카드 레벨에서 공간 부족으로 제외

---

## 3. 주요 결정 사항

### Company Description 구조
- 시스템이 웹검색으로 자동 생성
- 구성: Summary + Business Portfolio (아이템명/아이템 설명)
- 매칭 방식: Semantic 매칭 (단순 키워드 아닌 문장 단위 의미 매칭)

### MVP 범위
- 태그 개수: 제한 없음
- 하이라이트 범위: Description 전체
- 하이라이트 상한: Description의 20%
- 매칭 없는 공고: 태그 없이 그대로 표시

### 성공 지표
- 전환율: 50% → 80%
