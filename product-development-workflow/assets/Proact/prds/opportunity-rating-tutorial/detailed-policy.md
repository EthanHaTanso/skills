## 1. 기능 개요

### 배경

유저는 온보딩 과정에서:
1. Company Profile을 입력/생성하고 확인(Confirm)
2. 시스템이 Relevant Opportunity를 로드 (5-10초 소요)
3. 로드된 Opportunity 중 "Interested" 여부를 Rating

**현재 문제**: Step 2→3 전환 중, 유저가 Opportunity 선별의 **맥락을 오해**합니다.
- **오해**: "내가 직접 수주할 프로젝트"를 골라야 함
- **실제 의도**: "일부라도 수행 가능한 프로젝트"를 고르면, 시스템이 함께 수주할 파트너를 찾아줌

**결과**: 전환율 50%, "나한테 맞는 게 없다"며 이탈 또는 잘못된 선택 기준 적용

**이번 기능의 역할**: 로딩 대기 시간을 활용하여 **순차 애니메이션 튜토리얼**을 제공, 유저가 다음 단계의 맥락과 의도를 학습하도록 함

### 핵심 가치

- **대기 시간의 가치 전환**: 5-10초 로딩을 교육 기회로 활용
- **올바른 멘탈 모델 형성**: "파트너와 함께 수주"라는 개념 전달
- **UI 요소 사전 학습**: Opportunity 카드 구조와 매칭 기능 미리 안내
- **전환율 향상**: 맥락 이해로 이탈 및 오선택 방지

## 2. 유저 상황과 기대

### 유저가 이 단계에 도달했을 때의 심리

- "Company Profile 확인했고, 이제 뭐가 나오는 거지?"
- "로딩 중인데 뭘 기다리는 거야?"
- "다음 단계에서 뭘 해야 하는 거지?"
- "이 서비스가 나한테 어떤 가치를 주는 건지 아직 잘 모르겠어"

### 유저에게 제공해야 할 경험

1. **대기 시간의 의미 부여** — 로딩이 지루하지 않고 유익한 시간이 됨
2. **명확한 과업 안내** — 다음에 무엇을 해야 하는지 미리 파악
3. **가치 제안 전달** — 왜 이 과업을 해야 하는지 (파트너 매칭) 이해
4. **UI 친숙화** — Opportunity 카드와 매칭 기능을 미리 학습하여 실제 화면 진입 시 자연스럽게 사용

## 3. 주요 UX 원칙

### 3.1. 로딩의 재정의 (Loading as Learning)

- 로딩 시간을 "기다리는 시간"이 아닌 "배우는 시간"으로 전환
- 튜토리얼이 로딩보다 길어도 OK — 유저가 Skip하지 않는 한 끝까지 진행
- **의도**: Perceived waiting time 감소 + 실질적 학습 효과

### 3.2. 시각 우선 (Visual-First)

| 접근 방식 | 문제점 | 우리의 선택 |
|-----------|--------|-------------|
| **텍스트 설명** | 읽기 싫음, 기억 안 남음 | ❌ |
| **Mock UI + 설명** | 실제와 연결됨, 직관적 | ✅ |

- 실제 Opportunity 카드 UI를 Mock 형태로 보여주며 설명
- 텍스트는 최소화, 시각적 요소(포인터, 하이라이트, 애니메이션)로 가이드
- **의도**: 인지 부하 최소화, 기억에 남는 학습

### 3.3. 순차적 집중 (Progressive Focus)

- 한 번에 모든 것을 설명하지 않음
- 3단계로 나누어 각 단계에서 하나의 개념에만 집중
- **의도**: 정보 과부하 방지, 단계별 이해도 향상

| Step | 집중 포인트 | 핵심 질문 |
|------|------------|-----------|
| Step 1 | 전체 맥락 | "왜 이걸 하는 거지?" |
| Step 2 | 카드 구조 | "뭘 보는 거지?" |
| Step 3 | 매칭 기능 | "어떻게 판단하지?" |

### 3.4. 탈출 가능 (Escapable)

- Skip 버튼 항상 표시
- 유저가 원하면 언제든 튜토리얼 건너뛰기 가능
- **의도**: 강제성 배제, 유저 통제감 유지

## 4. 상세 유저 플로우

### 4.1. 튜토리얼 진입 — 로딩 화면 전환

**상황**
- 유저가 Company Profile "Confirm" 버튼 클릭
- 시스템이 Relevant Opportunity 로딩 시작 (API 호출)
- 기존: 스피너 + "Loading matching opportunities..."

**플로우**
1. Confirm 버튼 클릭 시:
   - 기존 스피너 대신 **튜토리얼 화면**으로 전환
   - 백그라운드에서 Opportunity 로딩 진행
2. 튜토리얼 화면 구성:
   - 중앙: 애니메이션 콘텐츠 영역
   - 우측 상단: "Skip" 버튼
   - 하단: Step indicator (1/3, 2/3, 3/3)
3. 첫 번째 Step 자동 시작

**디자인 의도**
- 로딩과 튜토리얼을 자연스럽게 연결
- Skip 버튼으로 강제성 배제

---

### 4.2. Step 1 — 페이지 개요 및 가치 전달

**상황**
- 유저가 튜토리얼 첫 화면을 봄
- "이게 뭐지?" 상태

**플로우**
1. **헤더 메시지** 표시:
   - "다음 단계에서 할 일을 알려드릴게요"
2. **Mock 페이지 레이아웃** 표시:
   - 전체 Opportunity 리스트 페이지의 축소 버전
   - 카드들이 나열된 모습 (블러 처리)
3. **핵심 메시지** 애니메이션으로 등장:
   - **"일부라도 수행할 수 있는 프로젝트를 고르세요"**
   - **"우리가 함께 수주할 파트너를 찾아드릴게요"**
4. 파트너 매칭 개념 시각화:
   - 간단한 다이어그램 또는 아이콘 애니메이션
   - [You] → [Select Opportunity] → [Find Partners]
5. 30초~1분 후 자동으로 Step 2로 전환 (또는 Next 버튼 클릭)

**디자인 의도**
- 핵심 가치 제안을 먼저 전달 ("왜 해야 하는지")
- "직접 수주"가 아닌 "파트너와 함께"라는 프레이밍

---

### 4.3. Step 2 — 기본 Opportunity 카드 구조

**상황**
- 유저가 "왜"를 이해했음
- 이제 "무엇을"을 배워야 함

**플로우**
1. **헤더 메시지** 업데이트:
   - "Opportunity 카드를 살펴볼게요"
2. **Mock Opportunity 카드** 표시:
   - 실제 데이터 기반 (가짜 데이터 X)
   - **매칭 태그/하이라이트 없는** 기본 버전
3. **순차적 요소 하이라이트**:
   - (1) **Title** 영역 강조 + 설명: "공고 제목이에요"
   - (2) **Agency** 영역 강조 + 설명: "발주 기관이에요"
   - (3) **Description** 영역 강조:
     - "프로젝트 내용이에요"
     - "Scope of Work에서 당신이 수행 가능한 부분을 찾아보세요"
4. 각 요소 설명 시 **포인터/화살표 애니메이션** 사용
5. 30초~1분 후 Step 3으로 전환

**디자인 의도**
- 실제 UI와 동일한 카드를 보여줘 학습 전이 효과 극대화
- 매칭 기능 없이 기본 구조만 먼저 소개 (Progressive Disclosure)

---

### 4.4. Step 3 — AI Curation 및 매칭 기능 소개 (v2 신규)

**상황**
- 유저가 카드 구조를 이해했음
- 이제 "어떻게" 판단하는지 배워야 함

**플로우**
1. **헤더 메시지** 업데이트:
   - "당신의 비즈니스와 연결된 부분을 찾는 법"

2. **기본 UI Mute 처리**:
   - 동일한 Mock 카드 사용
   - Title, Agency, Description 텍스트 Mute (opacity 0.3-0.5)
   - **Rating 버튼은 Mute하지 않음**

3. **CurationBadge 등장** (애니메이션):
   - 카드 우측 상단에 녹색 "✓ Match" 뱃지 등장
   - Fade in + Scale up 애니메이션
   - 설명: **"이 뱃지가 보이면 AI가 당신의 비즈니스와 맞다고 판단한 거예요"**

4. **Portfolio 태그 등장**:
   - 카드 상단에 Business Portfolio 매칭 태그 표시
   - 예: `Cloud Computing` `Infrastructure`
   - 설명: **"이 태그는 당신의 Business Portfolio와 매칭된 영역이에요"**

5. **Description 하이라이트 등장**:
   - Description 내 특정 문장에 노란색 하이라이트
   - 설명: **"하이라이트된 부분이 당신의 사업과 연결돼요"**

6. **RationaleCallout 등장**:
   - 카드 상단(Title 아래)에 연두색 배경 Callout 등장
   - 내용: "• Cloud infrastructure와 연관\n• IT 컨설팅 역량 활용 가능"
   - 설명: **"AI가 왜 맞다고 판단했는지 이유도 알려드려요"**

7. **"Curated for You" 체크박스 소개 (인터랙티브)**:
   - 화면 상단에 체크박스 UI Mock 표시 (초기: 미선택 상태)
   - 설명: **"여기를 클릭하면 AI가 찾은 Opportunity만 볼 수 있어요"**
   - **유저가 직접 클릭하도록 유도** (포인터 + 하이라이트)
   - 유저가 클릭하면:
     - 체크 애니메이션 + 카드 필터링 시뮬레이션
     - **선택 상태 저장** → 실제 Opportunity 페이지 진입 시 "Curated for You" 체크된 상태로 제공
   - 클릭 안 하고 넘어가면:
     - 기본 상태(미선택)로 실제 페이지 진입

8. **행동 가이드**:
   ```
   ✓ "Match" 뱃지가 있는 공고를 우선 확인해보세요
   ✓ 태그와 하이라이트로 어떤 부분이 연결되는지 빠르게 파악하세요
   ✓ Rationale(이유)를 읽으면 더 정확하게 판단할 수 있어요
   ✓ 모두 보고 싶으면 "Curated for You" 체크박스를 해제하세요
   ```

9. **최종 CTA**:
   - "시작하기" 버튼
   - 로딩 완료 시 즉시 전환, 미완료 시 대기 화면

**디자인 의도**
- LLM Curation의 핵심 UI 요소를 모두 시연 (Badge + Tag + Highlight + Rationale)
- **체크박스 직접 클릭**으로 학습 효과 + 실제 페이지 설정 연동
- "AI가 도와준다"는 인식으로 탐색 부담 감소

---

### 4.5. 튜토리얼 완료 및 전환

**상황**
- Step 3 완료
- Opportunity 로딩이 완료되었거나 진행 중

**플로우**
1. **로딩 완료된 경우**:
   - "시작하기" 버튼 클릭 시 즉시 Opportunity 리스트 화면으로 전환
   - Step 3에서 "Curated for You" 체크했으면 → 체크된 상태로 진입
2. **로딩 미완료인 경우**:
   - 대기 화면 표시:
     - 로딩 스피너 + "거의 다 됐어요..."
     - **"튜토리얼 다시 보기" 버튼** (Secondary)
   - 버튼 클릭 시 튜토리얼 Step 1부터 다시 시작
   - 로딩 완료 시 자동 전환

**디자인 의도**
- 튜토리얼과 로딩의 싱크 처리
- 대기 시간에 튜토리얼 복습 옵션 제공
- 체크박스 설정 연동으로 학습 → 실제 사용 자연스럽게 연결

---

### 4.6. Skip 처리

**상황**
- 유저가 튜토리얼 도중 Skip 버튼 클릭

**플로우**
1. Skip 버튼 클릭 시:
   - 튜토리얼 즉시 종료
   - 로딩 완료 상태면 즉시 리스트 화면 ("Curated for You" 미선택 상태)
   - 로딩 미완료면 대기 화면 (+ "튜토리얼 다시 보기" 버튼)
2. Skip 여부 기록 (Analytics)

**디자인 의도**
- 유저의 선택권 존중
- Skip 데이터로 튜토리얼 개선 인사이트 수집

## 5. 엣지 케이스 및 예외 처리

### 5.1. 로딩이 튜토리얼보다 먼저 완료

- **상황**: API 응답이 3초 내 완료 (튜토리얼 진행 중)
- **처리**: 튜토리얼 끝까지 진행, 완료 후 즉시 전환
- **복구 옵션**: Skip 버튼으로 언제든 종료 가능

### 5.2. 로딩 실패

- **상황**: Opportunity 로딩 API 에러
- **처리**:
  - 튜토리얼 종료 후 에러 화면 표시
  - "다시 시도" 버튼 제공
- **메시지**: "Opportunity를 불러오지 못했어요. 다시 시도해주세요."

### 5.3. 페이지 새로고침

- **상황**: 튜토리얼 도중 새로고침
- **처리**:
  - Company Profile 상태가 저장되어 있으면 → 튜토리얼 처음부터 다시 시작
  - 저장되지 않았으면 → 온보딩 처음으로
- **"Curated for You" 상태**: 새로고침 시 초기화됨 (미선택 상태)

### 5.4. Mock 데이터 미준비

- **상황**: 실제 데이터 기반 Mock이 준비되지 않은 경우
- **처리**: 사전에 선정된 대표 Opportunity 데이터 사용 (하드코딩)
- **Curation Mock**: CurationBadge, Portfolio 태그, RationaleCallout 예시 데이터 포함

### 5.5. "Curated for You" 체크박스 상호작용

- **상황**: Step 3에서 유저가 체크박스를 클릭하지 않고 넘어감
- **처리**: 기본 상태(미선택)로 실제 Opportunity 페이지 진입
- **상황**: 유저가 체크박스를 클릭함
- **처리**: 선택 상태 localStorage에 저장 → 실제 페이지에서 "Curated for You" 체크된 상태로 초기화

## 6. 향후 고려사항

### 6.1. 튜토리얼 A/B 테스트

- **현재**: 전체 유저에게 동일 튜토리얼 제공
- **향후**: 튜토리얼 유무/길이별 전환율 비교 테스트

### 6.2. 튜토리얼 다시보기

- **현재**: 로딩 대기 중 "튜토리얼 다시 보기" 버튼 제공
- **향후**: 설정 또는 도움말에서 다시 볼 수 있는 옵션 검토

### 6.3. 단계별 인터랙티브 요소

- **현재**: Step 3의 "Curated for You" 체크박스만 인터랙티브
- **향후**: 더 많은 인터랙티브 요소 검토 (예: Rating 버튼 클릭 시연)

### 6.4. 개인화된 튜토리얼

- **현재**: 모든 유저에게 동일 콘텐츠 (Portfolio 기반 예시는 동적)
- **향후**: 유저의 업종/경험에 따라 강조점 차별화

### 6.5. Curation 확장

- **현재**: 온보딩 튜토리얼에서 Curation 기능 소개
